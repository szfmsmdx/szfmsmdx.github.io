A4

VGG

！利用多个小的卷积核堆积达到一个大核的效果：在感受野相同的情况下，减少所需要的参数数目

例如，文章指出 三个 3x3 的卷积核效果等价于一个 7x7 的卷积核 \[ 7 \times 7 \times C \times C = 49 C^2 \\ 3 \times (3 \times 3 \times C \times C) = 27C^2 \] 所需要的参数是多个小核更小。。。

R-CNN

参考了这篇文章

R-CNN的R代表的是 region，利用了 region proposal 算法，生成候选区域，主要的流程为：

  1. region proposal，获取候选区域
  2. 获取图像特征，将图像输入到卷积神经网络中获取图像特征
  3. 获取区域类别，在初步获得目标的位置之后，需要获取目标的类别
  4. 微调区域位置，尽管候选区域已经初步目标的位置，但是这个区域比较粗糙，因此使用回归器对区域位置进行微调

1. selective search

参考博客

  1. 计算所有邻近区域之间的相似性；
  2. 两个最相似的区域被组合在一起；
  3. 计算合并区域和相邻区域的相似度；

重复2、3过程，直到整个图像变为一个地区。

2. 提取特征、获取区域类型

将分割得到的区域缩放到同一大小，然后送入预先训练好的 CNN 网络，拓展维度，注意 CNN 最后的全连接层给去掉，得到一个维度矩阵，将该维度矩阵送入 SVM 分类器判断各个类别

size 变化如下：

候选区个数 2000， reshape -> 227 x 227，此时的 tensor shape 为 (2000, 227, 227)

送入 CNN，提取维度为 4096，此时 tensor shape 为 (2000, 4096)

送入 SVM ，20个SVM分类器，每个权重阵大小为 2096，就得到 2000 个候选区的类别得分，此时shape 为 (2000, 20)

3. NMS

Non-Maximum Suppression，NMS，非极大值抑制，利用贪心策略对可能重复定位的候选区进行合并

  1. 我们对所有检测到的边界框进行排序，排序依据是每个边界框的置信度得分（即物体存在的概率）。
    1. 选择置信度最高的边界框，并从候选列表中移除它。
    2. 计算其与剩余候选框的 IoU，高于预设定阈值认为是重复检测了，将他们删掉
    3. 重复循环内的步骤1，直到无候选框
  2. 输出

优缺

  1. 优点

    1. 准确性：RCNN在目标检测任务中表现出很高的准确性。通过使用深度学习模型（卷积神经网络）提取图像特征，RCNN能够学习到更具判别性的特征表示，从而提高了目标检测的准确性。

    2. 候选区域生成：RCNN采用选择性搜索（Selective Search）算法来生成候选区域。相比传统方法的密集滑动窗口搜索，选择性搜索能够快速生成大量可能包含目标的候选区域，减少了计算开销。

    3. 端到端训练：RCNN采用端到端的训练方式，能够同时学习特征提取和目标分类器的参数。这使得RCNN能够端到端地优化整个目标检测系统，提高了检测性能。

    4. 可扩展性：RCNN框架具有良好的可扩展性，可以方便地应用于不同的目标检测任务和数据集。通过改变网络结构和训练数据，RCNN可以适应不同的目标类别和场景。

  2. 缺点：

    1. 低速：RCNN算法的一个主要缺点是计算速度较慢。在目标检测过程中，需要为每个候选区域进行特征提取和分类，这导致了大量的计算量和时间消耗。

    2. 复杂性：RCNN算法的实现相对复杂。它包含了多个组件，包括候选区域生成、特征提取、分类器训练等。这使得算法的实现和调优相对困难。

    3. 内存消耗大：RCNN算法需要在内存中存储大量的候选区域和特征表示，导致内存消耗较大。

    4. 训练时间长：由于RCNN的复杂性和计算需求，训练RCNN模型需要大量的时间和计算资源。

在一开始 region proposal 的时候就已经生成很多冗余、重复的区域了。。。不知道有没有算法可以从一开始进行剔除，这样效率应该会高很多

Fast R-CNN

将 RCNN 的三大步骤分为两个小部分

  1. 候选区域生成方法不变，即selective search算法随机采样获得候选框。
  2. 将尺寸不一的整幅图输入cnn，获得特征图；将候选区域映射到特征图feature map中得到特征矩阵
  3. 特征矩阵通过ROI Pooling层（最大池化下采样，不限制输入尺寸大小）缩放到固定尺寸的特征图，展平+两个全连接得到RoI feature vector；然后并行分别操作全连接层，一个使用softmax loss分类器得到类比概率，一个bbox边界框回归器训练。

部分讲解

  1. ROI Pooling

给定输出大小，将候选区进行划分，针对每个块执行最大池化下采样操作，使得特征映射上不同大小的候选区域变为尺寸大小的特征向量，然后传入下一层。这样就不限制输入图像的特征了。。。

可以看这篇博客

  1. SPP Net

由于可能给出非常多的候选框，因此可以取一部分，比如前 64 个，那么现在就是一张图片生成了 64 个大小不一的候选框，现在的问题是，如何将这些选框的大小调整统一

上述图片的意思是生成了256个feature map，然后对这些 feature map 做 pooling，得到 21 = 4x4 + 2x2 + 1x1 的向量，一共就是 (256, 21)，这里输出就被固定住了

优劣

  1. 优点：
    1. fast RCNN 是先卷积，然后将各个候选框映射到卷积后的结果中得到 feature map，避免了 RCNN 中直接选框然后一个一个卷积提取特征
    2. fast RCNN使用的 ROI pooling技术避免了将图像进行拉扯，损失信息
  2. 缺点：
    1. 瓶颈局限在 selective search上，如何快速的提出 region proposal ？

FCOS

文章第一个要复现的是 one stage的模型——FCOS， Fully Convolutional One-Stage Object Detection

  • 输入端：image，大小为 H x W
  • 特征提取：FPN，，对每个像素点进行多尺度预测
  • Head：五个head共享权重（文章说对精度几乎无影响），并有三个分支
    • classification：目标分类，(H,W,C)
    • centerness：中心度，(H,W,1)
    • regression：目标尺寸，(H,W,4)

对于目标区域 bbox ，每个像素点都具有回归目标，即每个像素坐标\((x,y)\) 的目标是\((l,r,t,b)\) ，其中 \(l、r、t、b\) 分别是该点到图像框 left、right、top、bottom 的距离

中心度 centerness 的计算公式为 \(\mathrm{certerness} = \sqrt{\frac{\min(l,r)}{\max{(l,r)}}\times\frac{\min{(t,b)}}{\max(t,b)}}\) ，引入中心度的作用为：

  • 在训练阶段：使用BCE loss，加入到代价函数中一起训练网络；
  • 在测试阶段：对预测的bbox进行打分：\(\mathrm{score}_{\mathrm{final}}=\mathrm{score}_{\mathrm{classification}}\times\mathrm{centerness}\) ，所低质量的bbox最后会通过NMS过滤掉。

如何训练？

  1. 将 image 送入网络提取特征获得 feature map
  2. 在 feature map中选取一些 像素点
  3. 将这些 像素点 映射回相应bbox，并且表明标签类别，如果处在多个bbox中，选择最小的那个
  4. （在推理中，多anchor通过 nms 算法进行剔除，最后剩一些预测概率大的框输出

由此可以发现，本质上还是滑窗生成窗口然后一个一个窗口训练（但是训练只有一步所以叫 one stage。。。

所以速度上还是慢的，精度上有一点点提升

Encoder-Decoder / Seq2Seq

特别的，还有一种是 n-to-m，输入、输出为不等长的序列。这种结构是Encoder-Decoder，也叫Seq2Seq，是RNN的一个重要变种。

Encoder： 语义向量c可以有多种表达方式，最简单的方法就是把Encoder的最后一个隐状态赋值给c，还可以对最后的隐状态做一个变换得到c，也可以对所有的隐状态做变换。

Decoder： 拿到c之后，就用另一个RNN网络对其进行解码，这部分RNN网络被称为Decoder。Decoder的RNN可以与Encoder的一样，也可以不一样。具体做法就是将c当做之前的初始状态h0输入到Decoder中：

  • 另一种做法是将c当作每一步的输入

当然，这些都是比较抽象的情况。。。

直白点说是两组单词序列之间的转化：

  • \(\rm{source} = <x1, x2, x3, .... , xn>\)
  • $ \rm{target} = <y1, y2, y3, ... , ym>$

所以Encoder得到了 \(C = \mathscr F(x_1, x_2, \cdots, x_n)\) ，对于 Decoder来说，就是根据历史信息和中间语义生成不同时刻的内容

\(y_i=\mathscr G (y_1,y_2,\cdots,y_{i-1},C)\)

总而言之，Seq2Seq不是某个具体模型，而是一种框架

  • Encoder：将 input序列 →转成→ 固定长度的向量
  • Decoder：将 固定长度的向量 →转成→ output序列
  • Encoder 与 Decoder 可以彼此独立使用，实际上经常一起使用

Encoder-Decoder 缺点

最大的局限性：

  • 编码和解码之间的唯一联系是固定长度的语义向量c

  • 编码要把整个序列的信息压缩进一个固定长度的语义向量c

  • 语义向量c无法完全表达整个序列的信息

  • 先输入的内容携带的信息，会被后输入的信息稀释掉，或者被覆盖掉

  • 输入序列越长，这样的现象越严重，这样使得在Decoder解码时一开始就没有获得足够的输入序列信息，解码效果会打折扣

因此，为了弥补基础的 Encoder-Decoder 的局限性，提出了attention机制。

Attention mechanism

参考了这篇文章

注意力机制（attention mechanism）是对基础Encoder-Decoder的改良。Attention机制通过在每个时间输入不同的c来解决问题

从上面的图来看，其实上面的 Encoder-Decoder模型是注意力不集中的：

\(y_1 = f(C)\) 、 \(y_2=f(y_1,C)\) 、\(y_3=f(y_1,y_2,C)\) 其中 f 是Decoder的非线性变换函数。

  • 不论那个单词输出，source的语义编码 C 都是一样的
  • 而语义编码C是由句子Source的每个单词经过Encoder

=》 编码产生的，这意味着不论是生成哪个单词，y1,y2还是y3，其实句子Source中任意单词对生成某个目标单词yi来说影响力都是相同的，这是为何说这个模型没有体现出注意力的缘由。这类似于人类看到眼前的画面，但是眼中却没有注意焦点一样。

举个例子：

I love you =》我 爱 你

在翻译“我”时，显然要更加考虑 I 这个单词，所以如果各个英文单词都是等权重的，就显得有些不合理了。。。

改进的方法也很简单，就是加上一些权重参数

那么这种概率该设置为多少呢？

由于 Decoder 采用 RNN 架构，因此在生成输出 \(y_i\) 时是知道前时刻 \(y_{i-1}\) 的输出的，而我们的目的就是希望得到输入 source 句子对 \(y_i\) 的权重影响的，那么可以用上一时刻的输出对各个单词做个比较，通过一个函数来获得他们之间对其的可能性，然后把 F 做一个 softmax 得到不同的概率即可

绝大多数Attention模型都是采取上述的计算框架来计算注意力分配概率分布信息，区别只是在F的定义上可能有所不同。下图可视化地展示了在英语-德语翻译系统中加入Attention机制后，Source和Target两个句子每个单词对应的注意力分配概率分布。

上述内容就是经典的Soft Attention模型的基本思想，那么怎么理解Attention模型的物理含义呢？一般在自然语言处理应用里会把Attention模型看作是输出Target句子中某个单词和输入Source句子每个单词的对齐模型，这是非常有道理的。

目标句子生成的每个单词对应输入句子单词的概率分布可以理解为：输入句子单词和这个目标生成单词的对齐概率，这在机器翻译语境下是非常直观的：传统的统计机器翻译一般在做的过程中会专门有一个短语对齐的步骤，而注意力模型其实起的是相同的作用。

Attention 机制的本质思想

如果把Attention机制从上文讲述例子中的Encoder-Decoder框架中剥离，并进一步做抽象，可以更容易看懂Attention机制的本质思想。

将 Source 中的构成元素想象成是由一系列的<Key,Value>数据对构成，此时给定Target中的某个元素Query，通过计算Query和各个Key的相似性或者相关性，得到每个Key对应Value的权重系数，然后对Value进行加权求和，即得到了最终的Attention数值。所以本质上Attention机制是对Source中元素的Value值进行加权求和，而Query和Key用来计算对应Value的权重系数。即可以将其本质思想改写为如下公式： \[ \rm {Attention(Query,Source)}=\sum_{i=1}^{L_x} Similarity(Query, Key_i) * Value_i \] 其中，Lx= ||Source|| 代表Source的长度，公式含义即如上所述。
